Description 

This server is a frontend for Rosetta, a software that focuses on the prediction and design of protein structures, protein folding mechanisms, and protein-protein interactions. We provide access to different applications in protein design that utilize backrub, a method for modeling of backbone flexibility [Smith and Kortemme, 2008]. There are four applications implemented at this time:

  • Model Flexibility: Creates an ensemble of structures by applying backrub to all residues.
  • Point Mutation: Creates an ensemble of structures with one or more mutations of specific residues and applies backrub to the neighborhood of the mutations. This method is described in [Smith and Kortemme, 2008].
  • Upload Mutation: Allows for a flexible definition of the residues subject to mutations and backrub. Instructions for creating a mutations file can be found in Data Formats.
  • Backrub ensemble: Creates a backrub ensemble according to "A correspondence between solution-state dynamics of an individual protein and the sequence and conformational diversity of its family" (submitted) [Friedland et al. 2009]



Tutorial  

To submit a new job log in and click on Submit in the second row of the menu at the top of the site. This opens the submission form, which is divided in three parts.

Top
Enter a name for your job that helps you finding it later and the structure that you want to model.

Middle -  General Settings 
Choose the version of Rosetta, the number of structures and whether the raw output should be stored or not.

Bottom -  Application Specific Settings 
Choose the application you want to use and enter the application specific parameters. An explanation of each parameter is shown when hovering the mouse over its name.

Once your done click the 'Sumbit' button on the bottom left. If a value is missing a popup window will notify you.

For each uploaded structure a number of independent simulations are made in order to create an ensemble of structures. Click on Queue in the menu to see the job queue. The table shows basic information about the jobs as their status (in queue, active, done, or error), who submited them, when they were submited, the job name, the kind of Rosetta used, the number of structures created and an error message for the highly unlikely case the simulation fails. Your username is highlighted in green to make it easier to indentify your jobs. To view more detailed information about a specific job, click on its entry in the table.

Depending on the status not all of the information described below is shown.

           Simulation data
  • The upper part shows the job name, its status and possible actions. If the job hasn't started yet it is possible to delete it from the queue by clicking on the delete link in the 'Actions' field.
  • In the middle following information is shown: computer that submitted the job, when it was submitted, started, and finished, it's computation time, the date when the data will be deleted from our database, and which kind of Rosetta was used.
  • The lower part of the table shows details about the simulation as for instance the application and the parameters of the simulation. In the green row, links for accessing the output are provided.
           Downloadable files

Files can be viewed individually or can be downloaded as a zip archive.

           Results

For certain applications an analysis of the raw output is automatically carried out. For these cases the results are visualized here. All the files shown are also included in the zip archive.

Parameter 

Number of structures  

The number of individual simulations that are carried out.

ROSETTA output  

This option allows the user to keep the raw Rosetta output as well as the resfile. Otherwise only the resulting pdb structures will be provided for download.


Data Formats  

User data must have the following formats.Please follow these formats and check your input before submitting to the server to avoid errors and wasting processor time.

PDB Complex  

Multi-chain protein complex using RCSB's PDB File Format.

If you submit a job that is using mini and it finishes with an "Rosetta Error", please make sure that the residue-ID's in the PDB-file are <1000.

Mutations List  

If this file is not given backrub is applied to all residues. This file contains a list of mutations to be made and residues that should be subject to the backrub move of a PDB Complex. If the residue is not given in this file it will be fixed.

4 space delimited columns representing:

  • Column 1. The Chain ID as in the PDB complex (case sensitive).
  • Column 2. The residue number as in the PDB complex.
  • Column 3. The backrub tag: B: use backrub, F: keep fixed
  • Column 4. The residue that should be used at this postition (one letter code). If more than one residue is given rosetta will pick the one with the lowest energy.
This means that three cases can be defined in this file. Case 0 is the default for all other residues (need not to be defined):
0.no backrub, no mutationA 111 F default
1.backrub w mutation A 111 B AFI
2.backrub w/o mutation A 111 B
3.mutation w/o backrub A 111 F AFI

Example File: 
A 10 B
A 11 B
A 12 B
A 13 B
A 14 B AFI
A 20 B
A 21 B
A 22 B
A 23 B
A 24 B
A 31 F AFI

Residues 10 to 14 and 20 to 24 are subject to the backbone move. For residues 14 and 31 a mutation with either Alanine (A), Phenylalanine (F) or Isoleucine (I) is allowed. Rosetta is going to pick the energetically most favorable amino acid for this position. For residue 31 the backrub move is not allowed.



Rosetta  

The Rosetta software focuses on the prediction and design of protein structures, protein folding mechanisms, and protein-protein interactions. Rosetta codes consistently exhibit repeated success in the Critical Assessment of Techniques for Protein Structure Prediction (CASP) competition as well as the CAPRI competition. Rosetta codes also address aspects of protein design, docking and structure. The software is the foundation for the Human Proteome Folding Project on the World Community Grid.

ROSETTA++ is the software suite that implements the algorithms used in folding, design and interaction modeling. It is currently licensed for free to users at academic and nonprofit institutions. Over 2000 academic users in more than 32 countries use Rosetta. Commercial entities can use Rosetta by paying a license fee. Revenue from licensing is reinvested in supporting continued software development.

miniRosetta is the new version of Rosetta that is currently developed and will be officially released soon. It takes full advantage of C++ and it's concepts.
